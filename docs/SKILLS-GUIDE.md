# Skills 选择指南

本指南帮助你快速找到适合的 skill 来完成写作任务。

## 快速决策

### 我需要...

**从零开始写正式文档**
- 使用：`formal-writing`
- 场景：报告、提案、公文、重要沟通
- 特点：完整的引导式工作流，包含上下文收集、迭代开发、读者验证

**快速写个邮件或笔记**
- 使用：`writing`
- 场景：邮件、笔记、草稿、简单文档
- 特点：轻量级、快速输出

**格式化已有内容为公文**
- 使用：`official-doc-format`
- 场景：已有文本内容，需要应用中国公文格式标准
- 特点：专注格式化，不修改内容

**去除文本的 AI 味道**（不推荐）
- 使用：`humanizer-zh`
- 场景：用户明确要求去除 AI 风格
- 代价：降低可读性和流畅性
- ⚠️ 不推荐用于正式文档

## 详细说明

### 工作流层（完整流程）

#### formal-writing - 正式文档写作

**何时使用：**
- 需要从零开始创建正式文档
- 文档需要经过审批或正式发布
- 希望通过引导式流程确保质量
- 需要验证文档对读者的有效性

**工作流程：**
1. **上下文收集** - 通过灵活的信息倾倒方式收集背景
2. **内容开发** - 逐节头脑风暴、筛选、起草、迭代
3. **读者验证** - 使用 sub-agent 测试文档有效性
4. **格式化交付** - 可选的格式化

**输入：**
- 文档类型（报告、提案、公文等）
- 目标受众
- 背景信息和关键要点

**输出：**
- 完整的正式文档
- 可选：Word .docx 格式
- 可选：应用公文格式标准

**特点：**
- 学习并应用用户偏好
- 迭代优化内容
- 验证文档对陌生读者的可理解性
- 可以调用 official-doc-format

**适用场景：**
- 季度报告、年度总结
- 项目提案、预算申请
- 政府公文、正式通知
- 决策文档、技术规范

---

#### writing - 快速写作

**何时使用：**
- 需要快速生成初稿
- 文档相对简单，不需要复杂流程
- 时间紧迫，需要立即输出

**工作流程：**
1. 理解需求
2. 直接生成初稿
3. 简单迭代（如需要）

**输入：**
- 写作需求
- 基本信息

**输出：**
- 初稿文档

**特点：**
- 轻量级流程
- 快速输出
- 适合简单场景

**适用场景：**
- 日常邮件
- 会议笔记
- 简单备忘录
- 快速草稿

---

### 工具层（单一功能）

#### official-doc-format - 公文格式化

**何时使用：**
- 已有文本内容
- 需要应用中国公文格式标准
- 需要生成 Word 文档

**功能：**
- 应用公文格式标准（红头、字体、间距等）
- 生成符合规范的 Word 文档
- 不修改内容，只处理格式

**输入：**
- 文本内容
- 公文类型（通知、报告、请示、函等）
- 格式要求（字体、页边距等）

**输出：**
- 格式化的 Word .docx 文件

**特点：**
- 专注格式化
- 支持中国公文标准
- 可被 formal-writing 调用

**适用场景：**
- 已有内容，需要套用公文格式
- 将 Markdown 转换为公文 Word 文档
- 批量格式化多个文档

---

### 特殊工具（按需使用）

#### humanizer-zh - 去除 AI 风格

**⚠️ 重要说明：**
这个 skill 的作用是**去除 AI 味道**，而不是润色或提升质量。使用后会降低可读性和流畅性。

**何时使用：**
- 用户明确要求"去除 AI 味道"
- 用户要求"让文本不那么像 AI 写的"
- 用户对 AI 风格有明确的反感

**不要使用的情况：**
- 正式文档写作
- 需要提升可读性
- 需要润色文本
- 默认的写作流程

**功能：**
- 移除 AI 特征性的表达方式
- 使文本更"人类化"

**代价：**
- 降低可读性
- 文本可能不连贯
- 流畅性下降

**输入：**
- 中文文本

**输出：**
- 去除 AI 风格的文本（可读性降低）

**特点：**
- 不集成到任何工作流
- 仅在明确要求时使用
- 不推荐用于正式文档

## 组合使用

### 场景 1：写一份正式的季度报告

```
1. 使用 formal-writing 创建内容
   - 收集背景信息
   - 逐节起草和迭代
   - 读者验证

2. 在 formal-writing 流程中选择输出格式
   - 选择 Word .docx 格式
   - 应用标准格式
```

### 场景 2：将现有文本转换为公文

```
1. 准备好文本内容

2. 使用 official-doc-format
   - 提供文本内容
   - 指定公文类型
   - 设置格式要求

3. 获得格式化的 Word 文档
```

### 场景 3：快速写一封邮件

```
1. 使用 writing
   - 说明邮件目的
   - 提供关键信息

2. 获得初稿
```

### 场景 4：用户要求去除 AI 味道（不推荐）

```
1. 确认用户真的需要去除 AI 风格
   - 提醒会降低可读性
   - 确认用户理解代价

2. 如果用户坚持
   - 使用 humanizer-zh
   - 提醒检查输出质量
```

## 常见问题

**Q: 我应该用 formal-writing 还是 writing？**

A: 看文档的重要性和复杂度：
- 重要、正式、需要审批 → formal-writing
- 简单、日常、快速输出 → writing

**Q: formal-writing 会自动格式化吗？**

A: 不会自动，但在流程中会询问你是否需要格式化。你可以选择：
- 输出 Markdown
- 输出 Word .docx
- 应用公文格式标准

**Q: 我可以只用 official-doc-format 吗？**

A: 可以，如果你已经有完整的文本内容，只需要应用格式。但如果需要从零开始写，建议用 formal-writing。

**Q: humanizer-zh 是用来润色的吗？**

A: **不是！** humanizer-zh 是用来去除 AI 味道的，会降低可读性和流畅性。如果你需要润色，应该：
- 在 formal-writing 的迭代过程中优化
- 或者手动修改文本
- 不要使用 humanizer-zh

**Q: 什么时候应该用 humanizer-zh？**

A: 只有当用户明确说"去除 AI 味道"、"让它不那么像 AI 写的"时才使用。正常的写作流程不需要它。

**Q: 我可以组合使用多个 skills 吗？**

A: 可以！formal-writing 可以调用 official-doc-format。但 humanizer-zh 不集成到工作流中，只在明确要求时独立使用。
