# Claude 工作区角色定义

## 目的

这个工作区为 Claude Code 提供专业化的角色系统，用于处理不同类型的任务。

## 核心角色

工作区包含以下专业角色，定义在 `claude/agents/` 目录：

### 规划者 (Planner)
- **文件**: `claude/agents/planner.md`
- **职责**: 任务分解、依赖分析、实施计划
- **使用场景**: 复杂任务需要分阶段执行时

### 写作者 (Writer)
- **文件**: `claude/agents/writer.md`
- **职责**: 内容创作、文档改写、格式调整
- **使用场景**: 需要专注于内容生产时

### 审查者 (Reviewer)
- **文件**: `claude/agents/reviewer.md`
- **职责**: 质量检查、正确性验证、改进建议
- **使用场景**: 需要独立审查和验证时

### 研究者 (Researcher)
- **文件**: `claude/agents/researcher.md`
- **职责**: 信息收集、来源验证、资料整理
- **使用场景**: 需要深入研究和信息收集时

## 角色协作

角色可以协同工作：

- **规划 + 写作**: 先规划结构，再执行写作
- **研究 + 写作**: 先收集资料，再创作内容
- **写作 + 审查**: 先创作草稿，再审查改进
- **规划 + 审查**: 先制定计划，再审查可行性

## 角色选择指南

### 何时使用规划者
- 任务复杂，需要分步骤执行
- 需要分析依赖关系
- 需要评估多种实施方案

### 何时使用写作者
- 需要创作新内容
- 需要改写现有文档
- 需要调整格式和风格

### 何时使用审查者
- 需要独立的质量检查
- 需要验证正确性
- 需要提供改进建议

### 何时使用研究者
- 需要收集背景信息
- 需要验证事实和来源
- 需要整理参考资料

## 与技能的关系

角色和技能是互补的：

- **角色** - 定义"谁"来执行任务
- **技能** - 定义"如何"执行任务

例如：
- 使用 `planner` 角色 + `planning` 技能 = 专业的规划执行
- 使用 `writer` 角色 + `formal-writing` 技能 = 专业的正式文档写作

## 扩展角色

添加新角色的步骤：

1. 在 `claude/agents/` 创建 Markdown 文件
2. 定义角色的职责和使用场景
3. 在此文件中更新角色列表
4. 在相关技能中引用新角色
