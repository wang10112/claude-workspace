# Planning 模块记忆

## 当前定位

- `planning` 已升级为 `project` 中的第四个正式模块
- 它负责把拆解、排序、决策闭环标准化

## 长期约束

- planning 输出必须显式列出 assumptions 与 unresolved decisions
- planning 不等于执行
- planning 要服务 shared canonical surface，而不是绕开它

## 维护规则

- shared 侧维护 workflow / skill / playbook / memory
- Claude / Codex 侧只做适配
- planning 结果应能直接交给 writing、review、verification 或 implementation work
