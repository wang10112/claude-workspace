# Formal Writing

## When To Use

- formal notices
- internal briefings
- explanation memos
- formatting-only handoffs
- Chinese polish after a draft is structurally correct

## Core Responsibility

Route the task to the correct sub-skill and keep drafting, polishing, and formatting as separate decisions.

## Input Expectations

- document type
- audience
- source facts
- any wording that must remain unchanged
- desired output form

## Output Expectations

- correct sub-skill activated for the stage
- factual discipline preserved throughout
- no stage conflated with another

## Boundary Rules

- do not draft and format in the same pass when they need separate decisions
- do not invent facts at any stage

## Shared Sources

- `shared/workflows/formal-writing.md`
- `docs/FORMAL-WRITING-MODULE.md`

## Routing

| Task | Sub-skill |
|------|-----------|
| General drafting | `skills/writing/` |
| Internal briefs and memos | `skills/internal-comms/` |
| Formatting-only | `skills/official-doc-format/` |
| Final Chinese polish | `skills/humanizer-zh/` |
| Role separation needed | `agents/writer.md` + `agents/reviewer.md` |
