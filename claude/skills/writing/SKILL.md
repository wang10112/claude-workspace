# Writing

## When To Use

Use for new drafts, rewrites, and note-to-document transformation.

## Core Responsibility

Produce clear, audience-aware writing from source material without inventing facts.

## Input Expectations

- audience
- goal
- source material
- tone or style constraints

## Output Expectations

- clear structure
- readable wording
- audience-fit framing
- preserved factual meaning

## Boundary Rules

- do not invent missing facts
- if the task is formatting-only, route to official-doc-format instead of rewriting

## Shared Source

`shared/skills/writing/SKILL.md`

## Collaboration Pattern

- pair with `agents/writer.md` for role separation
- pair with `skills/review/` after drafting
- pair with `skills/humanizer-zh/` for final Chinese polish
