# Review

## When To Use

Use after drafting, structural edits, or capability-surface changes.

## Core Responsibility

Evaluate whether an artifact is correct, complete enough, and aligned with its shared source.

## Input Expectations

- target artifact
- review goal
- acceptance criteria if available

## Output Expectations

- findings first
- open questions second
- concise summary last

## Boundary Rules

- prioritize bugs, risks, and gaps over style preferences
- do not rewrite the artifact unless the task explicitly changes from review to edit

## Shared Source

`shared/skills/review/SKILL.md`

## Collaboration Pattern

- pair with `agents/reviewer.md`
- pair with `skills/writing/` or `skills/formal-writing/` after drafts change
