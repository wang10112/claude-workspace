# Review Workflow

## Purpose

Provide a neutral workflow for reviewing drafts, capability changes, and framework updates before they are considered ready.

## Typical Inputs

- target artifact
- review goal
- acceptance criteria
- relevant context or changed scope

## Typical Outputs

- findings-first review
- open questions
- release or next-step recommendation

## Capability Lanes

- review
- verification when readiness needs explicit checks
- research when standards or current references matter

## Suggested Shared Skill Route

- `review`
- `verification`
- `research` when external references are required
