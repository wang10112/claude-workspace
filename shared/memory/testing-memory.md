# 测试模块记忆

## 模块概述

测试模块负责软件质量保证，包括单元测试、集成测试、E2E测试和测试覆盖率分析。

## 核心原则

1. **测试优先** - 先写测试，再写代码（TDD）
2. **快速反馈** - 测试应该快速执行
3. **独立性** - 测试之间相互独立
4. **可重复** - 测试结果可重复
5. **自动化** - 测试自动运行和验证

## 测试策略

### 测试金字塔
- 70% 单元测试 - 快速、大量、细粒度
- 20% 集成测试 - 中等速度、适量、模块级
- 10% E2E 测试 - 慢速、少量、系统级

### 覆盖率目标
- 核心业务逻辑：90%+
- 工具函数：80%+
- UI 组件：70%+

## 常用框架

### JavaScript/TypeScript
- Jest - 全功能测试框架
- Vitest - 快速的 Vite 原生测试
- Cypress/Playwright - E2E 测试

### Python
- pytest - 功能强大的测试框架
- unittest - Python 标准库

### Go
- testing - Go 标准库
- testify - 断言和 mock 库

## 最佳实践

### 测试命名
```
test_<function>_<scenario>_<expected>
```

### 测试结构（AAA）
```
Arrange - 准备测试数据
Act - 执行操作
Assert - 验证结果
```

### 测试隔离
- 每个测试独立运行
- 使用 setup/teardown
- 避免测试间依赖
- 使用测试数据库或 mock

## 常见问题和解决方案

### 问题：测试太慢
**解决方案**：
- 使用并行执行
- Mock 外部依赖
- 优化测试数据
- 使用内存数据库

### 问题：测试不稳定
**解决方案**：
- 避免时间依赖
- 清理测试状态
- 使用确定性数据
- 增加适当等待

### 问题：覆盖率不足
**解决方案**：
- 识别未测试代码
- 补充边界测试
- 添加异常测试
- 测试错误路径

## 工作流程

1. **分析需求** - 理解要测试的功能
2. **设计测试** - 编写测试计划和用例
3. **编写测试** - 实现测试代码
4. **执行测试** - 运行测试并分析结果
5. **维护测试** - 更新和优化测试

## 质量门禁

- 所有测试必须通过
- 覆盖率不能降低
- 新代码必须有测试
- PR 必须包含测试

## 持续改进

### 定期审查
- 每月审查测试质量
- 识别脆弱的测试
- 更新过时的测试
- 删除无效的测试

### 重构测试
- 提取公共测试工具
- 使用测试 fixture
- 优化测试性能
- 改进测试可读性

## 相关模块

- `debugging` - 调试失败的测试
- `refactoring` - 重构测试代码
- `review` - 审查测试质量
- `verification` - 验证测试覆盖率
