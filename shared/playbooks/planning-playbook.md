# Planning Playbook

## Purpose

Turn recurring planning work into a reusable workflow for decomposition, sequencing, and decision closure.

## Default Planning Order

1. Define the goal and current state.
2. List constraints, dependencies, and hidden assumptions.
3. Break work into phases with clear sequencing.
4. Identify unresolved decisions explicitly.
5. End with success criteria and next actions.

## Planning Output Shape

- goal
- current state
- phased plan
- assumptions and open questions
- success criteria

## Good Fit

- multi-step implementation work
- workspace recovery
- capability expansion
- structural refactors

## Bad Fit

- pretending the work is fully specified when it is not
- turning straightforward execution into unnecessary planning overhead
