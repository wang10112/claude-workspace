# Review Playbook

## Purpose

Turn recurring review work into a reusable, low-drift workflow inside the shared workspace.

## Default Review Order

1. Confirm the review target and acceptance criteria.
2. Check correctness, risks, and missing validation first.
3. Distinguish factual or structural issues from style preferences.
4. State what was reviewed and what was not reviewed.
5. End with a clear recommendation:
   - ready
   - ready with minor follow-ups
   - not ready

## Review Output Shape

- findings first
- open questions second
- concise summary last

## Good Fit

- shared skill changes
- adapter drift review
- document review before publishing
- framework change review

## Bad Fit

- rewriting the target when the task is review-only
- claiming readiness without naming missing checks
